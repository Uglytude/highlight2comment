> 施工 AI(Sam / Codex)动手前必读。这是纪律,违反 = 返工。

## 这是什么
一个极简 Chrome 扩展(Manifest V3)。读网页时选中一句话、写一句评论,存成 Markdown,永久累积在本地,一键导出、并可自动追加进 Obsidian 库。开源、免费、纯本地、无账号、无云、无服务器。

## 谁在干活
- 施工(写代码):**Sam(Codex)**,就是你。
- 监理(定方案 / 验收 / 写文档):Claude(妙妙的助手)。
- 甲方:妙妙,不写代码的产品经理,ADHD,跟她说话要说人话。
- **同一时间,这个仓库只有一个 AI 施工。**

## 硬性规则(违反 = 返工)
1. **先读 `BUILD_SPEC.md` 再动手**,那是图纸。有不确定就在输出里写清疑问,不许瞎猜。
2. **只做 V1 范围**。任何"顺手加的功能"一律不要(YAGNI / KISS)。明确禁止:颜色、页面高亮留痕、账号、登录、云同步、AI 摘要、后端服务器。
3. **代码要对 AI 友好**:
   - 单一职责:每个文件、每个函数只做一件事;文件名说啥,里面就放啥。
   - 函数尽量短(30 行是提醒线)。多步操作时,主函数只做调度,每步拆成独立小函数。
   - 分层:底层(storage / markdown / writer)不许知道上层(UI)长什么样。
4. **第一天就装黑匣子**:`src/lib/logger.js` 足迹日志。关键动作(存一条、写文件、授权、导出、报错)都打点,方便 debug。
5. **快速失败**:出错就明确报错并记日志,绝不静默吞错、绝不悄悄丢数据。铁律:**先把本地存好,再尝试写 Obsidian**;写 Obsidian 失败不能影响本地已存的数据。
6. **不删用户数据**。任何清空 / 删除操作必须用户明确点确认才执行。
7. **Git 走 PR,不直接推 main**。commit 信息说清改了什么。
8. **安全**:任何 token / 密钥都不许写进代码或 remote URL;不上传任何用户内容到任何地方(本项目没有服务器)。

## 完工的定义
见 `BUILD_SPEC.md` 末尾《验收清单》。每一条都要**实际跑通**,不是"应该能跑"。自检完把结果写出来。
